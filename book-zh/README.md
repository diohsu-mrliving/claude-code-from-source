---

當 Anthropic 將 Claude Code 發布到 npm 時，`.js.map` source map 中的 `sourcesContent` 欄位包含了完整的原始 TypeScript 程式碼。本書即是研究該架構的成果，將其中的模式、取捨與設計決策提煉為任何工程師都能學習的技術敘事。

**共 18 章，分為 7 個部分。** 印刷版約 400 頁。

每一章都有層次遞進的深度：面向技術領導者的敘事流程、面向實作者的深度探討，以及一個名為「**應用這些**」的結尾，提煉出你可以直接借用到自己系統中的可轉移模式。圖表使用 [Mermaid](https://mermaid.js.org/) 繪製，可在 GitHub 上原生渲染。

## 讓一切運作的 10 個模式

如果你只讀這一段：

1. **AsyncGenerator 作為代理迴圈** — 產生 Messages、有型別的 Terminal 回傳值、天然的背壓與取消機制
2. **投機工具執行** — 在模型串流期間、回應完成之前，就開始執行唯讀工具
3. **並發安全批次處理** — 依安全性分區工具，讀取並行執行，寫入序列化
4. **Fork 代理以共享快取** — 平行子代理共享位元組完全相同的 prompt 前綴，節省約 95% 的輸入 token
5. **4 層 context 壓縮** — snip、microcompact、collapse、autocompact——每一層都比下一層輕量
6. **以檔案為基礎的記憶搭配 LLM 召回** — Sonnet 子查詢選取相關記憶，而非關鍵字比對
7. **兩階段技能載入** — 啟動時只載入 frontmatter，呼叫時才載入完整內容
8. **Sticky latch 維持快取穩定性** — 一旦發送某個 beta header，整個 session 期間絕不取消設定
9. **Slot 預留** — 預設 8K 輸出上限，命中時升級至 64K（在 99% 的請求中節省 context）
10. **Hook 設定快照** — 啟動時凍結，防止執行期注入攻擊

## 本書的製作過程

原始碼從 npm source map 中提取。36 個 AI 代理分四個階段分析了近兩千個 TypeScript 檔案：

1. **探索**：6 個平行代理讀取原始碼樹中的每一個檔案
2. **分析**：12 個代理撰寫了 494KB 的原始技術文件
3. **寫作**：15 個代理從零開始將所有內容改寫為敘事章節
4. **審閱與修訂**：3 位編輯審閱者提出了 900 行回饋意見；3 個修訂代理套用了所有修正

整個流程——從原始碼提取到最終修訂版完成——大約耗時 6 小時。
